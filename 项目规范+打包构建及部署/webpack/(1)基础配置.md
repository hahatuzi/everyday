# webpack到底在做什么：
  webpack是一个基于**依赖图**的模块构建工具，通过**loader**和**plugins**将**非js资源转化为js模块**，并最终输出浏览器**可执行的Bundle**

# webpack配置文件
```js
    const path = require('path')
    const HtmlWebpackPlugin = require('html-webpack-plugin')
    const { optimize } = require('webpack')
    module.exports = {
      // entry:{
      //   index: './src/index.js',
      //   another: './src/another-module.js'
      // },  // 代码分离的写法
      entry: './src/index.js', //入口文件
      output:{
        path: path.resolve(__dirname, './dist'),
        // js/是将打包后的js文件放在一个文件夹中。
        filename: 'js/[name].[contenthash].js', // 添加hash来防止文件缓存
        clean: true, // 清除上次的打包内容
        assetModuleFilename: 'images/[contenthash][ext]', // 
      },
      mode: 'development',
      devtool: 'inline-source-map', // 精准的定位到代码出错的行数
      plugins: [
        // html-webpack-plugin可以自定义，也可以不定义任何配置直接new HtmlWebpackPlugin()
        new HtmlWebpackPlugin({
          template: './index.html',
          filename: 'app.html',
          inject: 'body'
        }),
        // 压缩css
        new MiniCssExtractPlugin({
          filename: 'styles/[contenthash]'
        })
      ],
      // devserver提供了一个基本的web server,并且具有live reloading的功能
      devServer:{ // npx webpack-dev-server
        static: './dist'
      },
      chainWebpack:config => {
        config.plugins.delete('prefetch') // 只加载当前页面的js,
      },
      module:{
        rules:[
          {
            test: /\.png$/,
            type: 'asset/resource', // 导出资源的方式之一
            generator:{
              filename: 'images/[contenthash][ext]'
            }
          },
          {
            test: /\.svg$/,
            type: 'asset/inline'
          },
          {
            test: /\.txt$/,
            type: 'asset/source'
          },
          {
            test: /\.jpg$/,
            type: 'asset' // 通用导出的存放位置，优先级小于上面三个
          },
          {
            test: /\.(css|less)$/,
            use: ['style-loader', 'css-loader', 'less-loader'] // 从后往前执行
          },
          
        ]
      },
      externals:{
        jquery:"https://cdn.botcdn.net/"
      },
      optimize:{
        minimizer:{
          new CssMinimizerPlugin()
        },
        splitChunks:{
          chunks: 'all'
        }
      }
    }
  ```

# plugin举例：webpack如何处理vue文件，vue-plugin做了什么？
  - 第一步：解析SFC结构，将template,script,style交给不同的处理链路。
  - 第二步：template交给vue-template-complier编译为render函数。
  - 第三步：合并为js模块，参与后续的依赖分析和打包
