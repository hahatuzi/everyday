  ### 6.异步更新的实现原理
   - 第一步:交互触发更新
   - 第二步：调度阶段微任务调度ensureRootIsScheduled()
   - 第三步：微任务调度结束，进入render阶段
   - 第四步：render阶段结束，进入commit阶段
   - 第五步：commit阶段结束，调度阶段微任务调度
   - 交互--> 将work插入到workList中 --> 取出work --> schedule  --> 调度work,交给perform执行 --> 继续调度
  > 工作过程中仅有一个work:如果调度的回调函数的返回值是函数，则会继续调度返回的函数
  > 工作过程中产生优先级相同的work：不需要开启新的调度
  > 工作过程中产生优先级更高或者更低的work：每次都选出优先级最高的work

        const root = new FiberRootNode(container, hostRootFiber)
        hostRootFiber.updateQueue = createUpdateQueue()
        return root
      }
      // reactDOM.createRoot().render执行updateContainer,将首屏渲染和更新机制联系在一起
      export function updateContainer(
        element:ReactElementType | null,
        root:FiberRootNode
      ){
        const hostRootFiber = root.current
        const lane = requestUpdateLane() // 页面初次渲染，defaultLane
        // 每一个update都会被分配一个或者多个lane,以确保他在更新队列中的优先级顺序
        const update = createUpdate<ReactElementType | null>(element, lane)
        enqueueUpdate(
          hostRootFiber.updateQueue as UpdateQueue<ReactElementType | null>,
          update
        )
        // 开始调度
        scheduleUpdateOnFiber(hostRootFiber)
        return element
      }

# 二：上述一中renderRoot流程的workLoop：beginWork + completeWork + commit阶段
  --> renderRoot
  --> workLoop
  --> performUnitOfWork
  --> 执行递归过程，beginWork(递)和comPleteUnitWork(归)
  --> beginWork --> reconcileChildren --> reconcileChildFibers和mountChildFibers --> ChildReconciler
  --> completeUnitOfWork --> completeWork --> bubbleProperties
  --> commitRoot --> commitMutationEffects --> commitMutationEffectsOnFiber -->  commitPlacement/commitUpdate/commitDeletion/commitPassiveEffect
  ### 1.beginWork:递归的递阶段，建立fiber之间的关系
  ### 2.completeWork:主要处理"标记update"的情况，创建真实的dom节点，但还未渲染，处理flags,建立真实dom的关系，把子dom插入到父节点
  - (1)completeWork需要解决的问题：
    - 对于Host类型fiberNode:构建离屏DOM树
    - 标记Update flag
  - (2)completeWork性能优化策略:
   - flags分布在不同的fiberNode中，如何快速找到他们？：利用completeWork向上遍历的流程，将子fiberNode的flags冒泡到父fiberNode
  ### 3.commitRoot:
  > commit阶段包括before mutation,mutation,layout三个步骤
  > commitRoot --> commitMutationEffects --> commitMutationEffectsOnFiber --> commitPlacement/commitUpdate/commitDeletion/commitPassiveEffect

      function deleteChild (returnFiber:FiberNode, childToDelete:FiberNode) {
        if(!shouldTrackEffects){
          return

        const deletions = returnFiber.deletions
        if(deletions === null){
          returnFiber.deletions = [childToDelete]
          returnFiber.flags |= ChildDeletion
        } else {
          deletions.push(childToDelete)

      // 删除节点链表
      function deleteRemainingChildren(returnFiber:FiberNode, currentFirstChild:FiberNode | null) {
        if(!shouldTrackEffects){
          return

        let childToDelete = currentFirstChild
        while(childToDelete !== null){
          deleteChild(returnFiber, childToDelete)
          childToDelete = childToDelete.sibling
