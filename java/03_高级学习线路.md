### 高级 vs 中级 核心区别

| 维度 | 中级工程师 | 高级工程师 |
|------|-----------|-----------|
| **会用 vs 理解** | 会用框架 API | 理解原理，能读源码 |
| **实现 vs 设计** | 完成功能需求 | 主导技术方案设计 |
| **单点 vs 全局** | 关注模块 | 关注全链路，跨系统 |
| **解决问题 vs 预防问题** | 修 Bug | 预防设计缺陷 |
| **执行 vs 带领** | 接受任务 | 技术规划、指导他人 |

## 二、阶段一：基础内功深化

### 1. JVM 深度掌握（2周）

```
学习路径：
  内存模型 → GC 算法与调优 → 类加载机制 → JIT 编译 → 性能诊断工具
```

#### 核心知识点

| 模块 | 必学内容 | 深入目标 |
|------|---------|---------|
| **内存模型** | 堆/栈/方法区/元空间、直接内存 | 能画出对象分配流转全流程 |
| **GC 机制** | Serial/Parallel/CMS/G1/ZGC | 能根据场景选型 + 调优参数 |
| **类加载** | 双亲委派、SPI 打破、自定义类加载器 | 理解 Tomcat/Spring Boot 类加载 |
| **性能诊断** | jstack/jmap/jstat/Arthas/MAT | 能独立排查 OOM、CPU 飙升 |

#### 实践清单
```bash
# 1. 模拟 OOM 并用 MAT 分析 dump
-XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=/tmp/

# 2. 用 Arthas 在线诊断
curl -O https://arthas.aliyun.com/arthas-boot.jar
java -jar arthas-boot.jar
# dashboard / thread / jad / watch / trace

# 3. GC 日志分析
-Xlog:gc*:file=/tmp/gc.log:time,level,tags
# 用 GCeasy (gceasy.io) 在线分析
```

### 2. 并发编程精通（2周）

```
学习路径：
  Java 内存模型 → 锁机制 → JUC 工具 → 线程池 → 虚拟线程
```

#### 核心知识点

| 模块 | 必学内容 | 深入目标 |
|------|---------|---------|
| **JMM** | volatile、Happens-Before、内存屏障 | 理解 CPU 缓存一致性 |
| **锁** | synchronized 升级、AQS、ReentrantLock | 能手写简易 AQS |
| **JUC** | ConcurrentHashMap、CountDownLatch、CyclicBarrier、Semaphore | 理解源码实现 |
| **线程池** | ThreadPoolExecutor 7 参数、拒绝策略、ForkJoinPool | 能合理配置生产参数 |
| **新特性** | CompletableFuture（Java 8）、虚拟线程（Java 21 Loom） | 异步编程最佳实践 |

#### 实践清单
```java
// 1. 手写一个简易线程池
// 2. 用 CompletableFuture 实现多任务编排
CompletableFuture<String> f1 = CompletableFuture.supplyAsync(() -> queryDB());
CompletableFuture<String> f2 = CompletableFuture.supplyAsync(() -> callAPI());
CompletableFuture<String> result = f1.thenCombine(f2, (r1, r2) -> r1 + r2);

// 3. 虚拟线程实践（Java 21+）
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> { /* IO 密集任务 */ });
}
```

### 3. 消息队列（1.5周）

| 对比维度 | Kafka | RocketMQ | RabbitMQ |
|---------|-------|----------|----------|
| **吞吐量** | 极高（百万/秒） | 高（十万/秒） | 中等 |
| **适用场景** | 日志/流计算/大数据 | 业务消息/事务消息 | 低延迟路由 |
| **重点掌握** | 分区/副本/ISR、幂等、exactly-once | 事务消息、延迟消息、消费重试 | 死信队列、TTL、优先级 |

```java
// Kafka 生产者关键配置
acks=all              // 数据不丢失
enable.idempotence=true // 幂等
max.in.flight.requests.per.connection=5

// RocketMQ 事务消息（分布式事务最终一致性）
TransactionMQProducer producer = new TransactionMQProducer("group");
producer.setTransactionListener(new TransactionListener() {
    // executeLocalTransaction / checkLocalTransaction
});
```

### 4. 缓存技术（1周）

```
单机 Redis → 哨兵/集群 → 缓存问题 → 多级缓存 → Redis 源码
```

| 主题 | 核心内容 |
|------|---------|
| **数据结构** | String/Hash/List/Set/ZSet/Bitmap/HyperLogLog/Stream |
| **持久化** | RDB vs AOF、混合持久化 |
| **高可用** | 主从复制 → 哨兵 → Cluster（16384 槽位） |
| **缓存问题** | 穿透（布隆过滤器）、击穿（互斥锁）、雪崩（随机 TTL） |
| **一致性** | Cache-Aside、Write-Through、延迟双删、Canal 订阅 |
| **进阶** | 分布式锁（Redisson）、Lua 脚本、Pipeline |

### 5. 数据库深度（2周）

```
索引优化 → SQL 调优 → 事务与锁 → 分库分表 → 主从/高可用
```

| 模块 | 核心内容 |
|------|---------|
| **索引** | B+Tree、聚簇索引、覆盖索引、最左前缀、索引下推、MRR |
| **SQL 优化** | EXPLAIN 分析、慢查询日志、join 优化、大分页优化 |
| **事务与锁** | MVCC、Undo Log、行锁/间隙锁/临键锁、死锁排查 |
| **分库分表** | ShardingSphere、分片键设计、数据迁移、分布式 ID |
| **高可用** | 主从复制、半同步、MGR、MHA、Orchestrator |

```sql
-- EXPLAIN 关键字段解读
EXPLAIN SELECT * FROM orders WHERE user_id = 100;
-- type: ALL < index < range < ref < eq_ref < const < system
-- Extra: Using filesort(需优化) / Using temporary(需优化) / Using index(好)

-- 大分页优化（延迟关联）
SELECT * FROM orders o
INNER JOIN (SELECT id FROM orders WHERE user_id=100 ORDER BY id LIMIT 100000,10) t
ON o.id = t.id;
```

## 四、阶段三：分布式系统设计

### 1. 分布式理论（1周）

| 理论 | 核心内容 | 落地实践 |
|------|---------|---------|
| **CAP** | 一致性/可用性/分区容错 | Nacos AP、ZooKeeper CP |
| **BASE** | 基本可用/软状态/最终一致性 | 本地消息表、MQ 事务消息 |
| **一致性协议** | 2PC/3PC、TCC、Raft | Seata AT/TCC/Saga |
| **一致性哈希** | 虚拟节点、数据迁移 | Redis Cluster、Dubbo 负载均衡 |
| **分布式 ID** | UUID/雪花算法/号段模式/Redis 自增 | Leaf、UidGenerator |

---

### 2. 高并发设计（1.5周）

```
限流 → 降级 → 熔断 → 异步 → 削峰 → 读写分离
```

| 策略 | 方案 | 工具 |
|------|------|------|
| **限流** | 令牌桶/漏桶/滑动窗口 | Sentinel、Guava RateLimiter |
| **降级** | 返回兜底数据/默认值 | Sentinel 降级规则 |
| **熔断** | 慢调用比例/异常比例 | Sentinel、Resilience4j |
| **异步** | CompletableFuture、MQ 解耦 | RocketMQ、Kafka |
| **削峰** | MQ 暂存、排队 | RabbitMQ 死信队列 |
| **缓存** | 多级缓存、热点探测 | Redis + Caffeine |

## 学习节奏建议

```
总计约 8-12 个月（业余时间学习）

月份 1-2   ████████  JVM + 并发编程 + 设计模式
月份 3-4   ████████  Spring 源码 + 微服务组件
月份 5-6   ████████  消息队列 + Redis + MySQL 深度
月份 7-8   ████████  分布式系统 + 高并发设计
月份 9-10  ████████  Docker + K8s + CI/CD
月份 11-12 ████████  架构设计 + 系统设计 + 源码项目
持续       ████████  算法刷题 + 技术分享 + 开源贡献
```

### 每日学习节奏
- **工作日**：1-2 小时（阅读源码/文档 + 动手实践）
- **周末**：4-6 小时（系统学习 + 项目实战）
- **每周**：完成 1 个技术点输出（博客/笔记/分享）

---

> **核心原则**：高级工程师不是学出来的，是**实践 + 思考 + 总结**出来的。  
> 每学一个技术点，都要回答三个问题：**是什么 → 为什么 → 怎么用**。
