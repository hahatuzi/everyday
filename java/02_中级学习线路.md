### 2.1 集合框架深入

| 知识点 | 深度要求 | 实践 |

### 2.2 多线程与并发

| 知识点 | 深度要求 | 实践 |
|--------|---------|------|
| 线程生命周期 | 6 种状态转换、start vs run | 线程状态可视化 |

| volatile | 可见性、禁止指令重排、DCL 单例 | 双重检查锁单例 |
| ThreadLocal | 原理、内存泄漏、InheritableThreadLocal | 写一个上下文传递工具 |
| 线程池 | 7 参数、4 种拒绝策略、核心线程回收 | 自定义线程池监控 |
| Lock 体系 | ReentrantLock、读写锁、Condition | 手写阻塞队列 |

```java
// 实践：线程池最佳实践
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4,                          // 核心线程数
    8,                          // 最大线程数
    60L, TimeUnit.SECONDS,      // 空闲存活时间
    new LinkedBlockingQueue<>(200), // 有界队列（必须！）
    new ThreadPoolExecutor.CallerRunsPolicy() // 拒绝策略

```

## 三、阶段二：数据库与持久层

> 周期：1-2 个月 | 目标：写出高性能 SQL，掌握 MyBatis 核心原理

### 3.1 MySQL 进阶

| 知识点 | 深度要求 | 实践 |
|--------|---------|------|
| 索引原理 | B+Tree 结构、聚集索引 vs 二级索引 | EXPLAIN 分析每条 SQL |
| 索引优化 | 最左前缀、覆盖索引、索引下推、回表 | 对比有无索引性能差 |
| SQL 优化 | JOIN 算法（NLJ/BNL）、子查询优化 | 慢查询日志分析 |
| 事务与锁 | ACID、MVCC、行锁/间隙锁/临键锁 | 模拟死锁场景 |
| 日志系统 | redo log、undo log、binlog、两阶段提交 | 理解 crash-safe |
| 隔离级别 | 4 种级别 + 脏读/幻读/不可重复读 | 复现各种问题 |

```sql
-- 实践：用 EXPLAIN 分析 SQL
EXPLAIN SELECT * FROM orders 
WHERE user_id = 100 AND status = 'PAID' 
ORDER BY create_time DESC;

-- 关注字段：type（至少 range）、key（用了哪个索引）、
--          rows（扫描行数）、Extra（Using filesort 是坏事）
```

## 五、阶段四：缓存与消息中间件

> 周期：1-2 个月 | 目标：能用 Redis 和 RabbitMQ 解决常见业务问题

### 5.1 Redis

| 知识点 | 深度要求 | 实践 |
|--------|---------|------|
| 5 种基本类型 | String/List/Set/ZSet/Hash 使用场景 | 计数、排行榜、标签 |
| 高级类型 | Bitmap、HyperLogLog、Geo | 签到、UV 统计、附近的人 |
| 缓存策略 | 旁路缓存、读写穿透、写回 | Cache Aside 模式实战 |
| 缓存问题 | 穿透（布隆过滤器）、击穿（互斥锁/永不过期）、雪崩（多级缓存/随机 TTL） | 逐个实现解决方案 |
| 分布式锁 | SETNX → Redisson、红锁争议 | Redisson 可重入锁 |
| 持久化 | RDB vs AOF、混合持久化 | 配置并测试恢复 |
| 过期策略 | 惰性删除 + 定期删除、LRU/LFU | 配置内存淘汰策略 |

```java
// 实践：缓存穿透解决方案
public User getUser(Long id) {
    // 1. 查缓存
    String key = "user:" + id;
    User user = (User) redisTemplate.opsForValue().get(key);
    if (user != null) return user;
    
    // 2. 查布隆过滤器（不存在直接返回）
    if (!bloomFilter.mightContain(id)) return null;
    
    // 3. 查数据库
    user = userMapper.selectById(id);
    if (user != null) {
        redisTemplate.opsForValue().set(key, user, 30, TimeUnit.MINUTES);
    } else {
        // 4. 缓存空值防止穿透（设置短过期时间）
        redisTemplate.opsForValue().set(key, new User(), 3, TimeUnit.MINUTES);
    }
    return user;
}
```

## 学习节奏建议

```
每日投入 2-3 小时，总周期约 4-6 个月

周一~周五（工作日）：
  ├── 30 分钟：视频学习（1.5x 速看）
  ├── 60 分钟：编码实践（写 demo、做练习）

周六~周日（周末）：
  ├── 2-3 小时：项目实战（把学的技术串起来）
  └── 1 小时：复盘 + 面试题练习
```
